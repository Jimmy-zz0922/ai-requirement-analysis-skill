# Case Study 01：房屋交易业务流程结构化分析

本案例演示如何使用 **AI Requirement Analysis Skill**，将一张跨系统房屋交易业务流程图转化为可追溯的流程摘要、问题候选、AI适配性判断和需求卡片。

> 为避免公开内部原始资料，本案例不包含原PDF，仅保留脱敏后的结构化摘要和必要证据描述。

## 案例目标

- 识别房屋交易全生命周期阶段
- 提取参与系统与关键业务节点
- 识别明确功能缺口与待验证的问题候选
- 判断问题应由AI、规则引擎还是普通系统功能解决
- 输出可供后续访谈与产品评审使用的需求卡片

## 输入材料

- 类型：业务流程图
- 页数：1页
- 主要内容：商品房预售许可、预售销售、首次登记、现售销售和二手房销售流程
- 涉及系统：住建局其他业务系统、房地产信息平台、不动产登记平台、测绘大队系统、规资局数据中心系统

## Skill处理过程

```text
材料登记
  → 阶段与系统识别
  → 关键节点抽取
  → 系统交互关系整理
  → 明确事实与分析判断分离
  → 问题候选生成
  → AI适配性评估
  → 需求卡片输出
  → 待确认事项整理
```

## 关键发现

1. 流程覆盖5个连续业务阶段，横跨5类业务系统。
2. 流程包含土地、规划、施工、测绘、交易、资金监管和不动产登记等多类业务信息。
3. 图中明确标记“现售楼盘表（未实现）”，可作为直接证据支持的功能缺口。
4. “跨系统状态同步与展示”“流程节点知识解释”属于分析形成的问题或产品假设，仍需通过访谈、系统清单和接口资料验证。
5. 对确定性流程、状态同步和楼盘表建设，应优先采用系统集成、数据治理或规则能力，而不是默认使用大模型。

## 文件说明

- `source-summary.md`：对原流程图的脱敏摘要
- `process-extraction.md`：业务阶段、系统与节点抽取结果
- `analysis-report.md`：问题诊断与AI适配性分析
- `structured-output.json`：符合Skill Schema的结构化需求卡片
- `validation-notes.md`：人工复核与下一步调研问题

## 使用边界

本案例仅基于一张流程图。它能够证明流程结构和图中明确标注的功能状态，但不能直接证明用户痛点、问题发生频率、实施优先级或业务成效。上述内容需要进一步调研和交叉验证。
