# Evaluation Rubric

每项0—2分，总分20分。

| 维度 | 0分 | 1分 | 2分 |
|---|---|---|---|
| 材料识别 | 未区分来源 | 部分区分 | 完整source_id、类型、定位 |
| 事实提取 | 大量遗漏或误读 | 基本正确 | 角色、场景、流程、规则完整 |
| 证据追溯 | 无证据 | 部分有证据 | 核心结论均有定位与原文 |
| 事实/判断/建议 | 混写 | 偶有混写 | 明确分层 |
| 痛点质量 | 把功能当痛点 | 表述一般 | 任务、阻碍、影响清楚 |
| 根因诊断 | 仅重复痛点 | 有初步根因 | 流程、系统、数据、机制清晰 |
| AI适配判断 | 默认AI优先 | 有边界意识 | 比较AI与非AI并说明理由 |
| 风险与复核 | 未说明 | 部分说明 | 高风险点、人工复核、回退完整 |
| 结构化输出 | 字段混乱 | 基本完整 | 符合Schema并可校验 |
| 待确认事项 | 无 | 泛泛而谈 | 精确指出材料缺口和决策问题 |

建议：
- 16—20：可进入真实项目试用
- 12—15：需补充证据或字段
- 0—11：需调整Skill指令或输入材料
